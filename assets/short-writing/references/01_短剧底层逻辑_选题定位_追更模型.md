# 短剧底层逻辑、选题定位与追更模型

## 短剧底层逻辑与追更模型

## 1. 本章解决什么

短剧不是慢热长故事的压缩版，而是以第一屏点击、前3集承诺、前10集追更理由和持续情绪回报为核心的故事性产品。写作优先级是：情绪入口、压迫位、反击工具、关系误判、证据/身份钩子、低成本可拍事件。

## 2. 短剧商业闭环

| 商业环节 | 剧作对应 | 失败症状 |
| --- | --- | --- |
| 点击 | 第一屏有强冲突、强关系、强身份差 | 开头是旁白、天气、回忆、世界观介绍 |
| 留存 | 每30-60秒有信息变化或权力变化 | 人物聊天但局势不变 |
| 追更 | 关键情绪未完成且下一集看似马上兑现 | 只是后来会怎样的弱好奇 |
| 投放 | 前3集有可剪切片段 | 卖点藏在20集以后 |
| 复看 | 阶段性回报持续出现 | 一直虐不爽，或一直爽无代价 |

## 3. 追更情绪拆解

| 追更情绪 | 观众等什么 | 最适合的结尾钩子 | 不适合的写法 |
| --- | --- | --- | --- |
| 打脸 | 低位者公开赢回尊严 | 证据即将公开或身份将被确认 | 主角太早完全赢 |
| 追悔 | 误判者发现自己错了 | 真相露一半，对方刚开始怀疑 | 误会没有代价 |
| 认亲 | 错位血缘回到正位 | 鉴定结果将出、被抢或被换 | 亲情只靠哭诉 |
| 复仇 | 伤害者付出对等代价 | 反派以为胜利，主角拿到关键证据 | 复仇对象太弱 |
| 甜宠 | 强关系从误解到偏爱 | 一方刚要保护另一方却被打断 | 甜而无压力 |
| 悬疑 | 真相被看见 | 线索指向熟人或亲密关系 | 信息太玄、问题不清 |

## 4. 短、快、强、密

- **短**：单集只解决一个剧作任务；一段对白最好只承担一个动作。
- **快**：开场不解释，先让人物处在压力里，再用冲突带信息。
- **强**：羞辱要公开，误会要有代价，反击要改变局面。
- **密**：每集至少包含事件推进、关系变化、信息释放、结尾钩子中的两项。

## 5. 追更点位置

| 集数段 | 主要任务 | 适合追更点 |
| --- | --- | --- |
| 1-3 | 建立压迫和承诺 | 主角身份或能力露出一角 |
| 4-10 | 证明持续好看 | 第一次强反击前被截断 |
| 11-20 | 放大关系和敌人 | 核心关系误判更深 |
| 21-40 | 中段规则改变 | 原反派背后还有人 |
| 41-60 | 秘密逼近公开 | 证据到手但公开会伤害重要关系 |
| 61-80/100 | 终局审判 | 所有人站队，真相即将公开 |

## 选题定位与赛道判断

## 1. 选题判断

短剧选题先判断“谁为哪种未完成情绪追更”，再判断题材。题材不是标签，而是承诺：霸总承诺权力差与偏爱，复仇承诺清算，追妻承诺迟来追悔，神豪承诺财富反差，家庭伦理承诺公开审判。

## 2. 选题评分表

| 维度 | 1分 | 3分 | 5分 |
| --- | --- | --- | --- |
| 一句话清晰度 | 讲不清主角和冲突 | 能讲清设定但卖点弱 | 一句话就有压迫、反击、关系或秘密 |
| 第一屏强度 | 需要解释背景 | 有冲突但不够尖锐 | 10秒内能看懂强情绪 |
| 赛道明确 | 多种类型混杂 | 主赛道清楚但副元素抢戏 | 主赛道明确，副元素服务主情绪 |
| 前10集持续性 | 只有一个梗 | 有几次升级但易重复 | 能自然铺出压迫、反击、秘密、关系倒置 |
| 可拍性 | 大场面多、成本高 | 可拍但场景分散 | 小空间强冲突，道具明确 |
| 差异化 | 完全同质 | 有一个新设定 | 情绪入口、身份关系或证据物件有新鲜感 |
| 边界风险 | 需要大幅规避 | 局部风险可调整 | 高刺激但边界清楚 |

25分以下先重做；25-30分补前10集；31分以上可进入提案。

## 3. 改编选题筛法

1. 保留最强压迫场，不保留慢铺垫。
2. 保留最有追更价值的秘密，不保留所有支线。
3. 保留关系旧账，不保留复杂人物谱系。
4. 把长线成长改成可见反击节点。
5. 把大场面改成证据、直播、饭局、办公室、医院、婚礼等可拍场。

## 4. 选题输出模板

```text

主赛道：
副元素：

第一屏冲突：

```


## 短剧开发总判断模型

短剧开发先问四个问题：谁被压、谁误判、谁会后悔、观众为什么必须看下一集。只要其中两个问题答不出来，项目就不应该进入分集阶段。

| 判断层 | 核心问题 | 可执行判断 |
| --- | --- | --- |
| 观众层 | 谁会为这个情绪追更 | 明确性别、年龄、关系处境、爽感偏好，不写泛泛大众 |
| 题材层 | 主赛道是什么 | 霸总、复仇、追妻、神豪、家庭伦理等只能有一个主承诺 |
| 事件层 | 第一集发生什么 | 必须能拍成一个公开冲突、证据冲突或关系选择 |
| 结构层 | 第10集卡在哪里 | 必须有强未完成，不是普通转场 |
| 制作层 | 能不能低成本拍 | 优先饭桌、病房、公司、婚礼、直播、车内、电梯、门口 |

## 追更情绪九宫格

| 情绪 | 观众爽点 | 常用题材 | 第一屏做法 |
| --- | --- | --- | --- |
| 被冤后反击 | 看坏人被公开打脸 | 复仇、家庭、职场、霸总 | 主角被当众定罪，但观众知道证据未出 |
| 错爱后追悔 | 看误判者付出代价 | 追妻、虐恋、婚姻 | 关系对象站错队，伤害具体发生 |
| 低位后翻身 | 看阶层或身份反差兑现 | 战神、赘婿、神豪、逆袭 | 低位身份被羞辱，高位资源露一角 |
| 血缘将揭 | 看错位家庭回到正位 | 萌宝、亲子、真假千金 | 鉴定、病历、称呼、旧照片出现 |
| 秘密将爆 | 看真相改变所有站队 | 悬疑、复仇、婚姻 | 证据被抢、直播将开、监控将放 |
| 甜宠补偿 | 看被偏爱和被保护 | 甜宠、霸总、萌宝 | 亲密推进被误会或外敌打断 |
| 制度压迫 | 看规则被反过来使用 | 古装、民国、家庭、职场 | 家法、合同、公司制度或舆论压主角 |
| 生存翻身 | 看小人物改变命运 | 年代、逆袭、都市爽文 | 钱、工作、名誉、孩子安全被威胁 |
| 危险逼近 | 看主角逼近真相 | 悬疑、惊悚、灵异 | 异常物件或失踪事件直接开场 |

## 选题开发流水线

```text
1. 写一句话故事：主角 + 低位 + 反击工具 + 追更问题。
2. 做题材归类：主赛道一个，副元素最多两个。
3. 写第一屏：10秒内让观众知道谁压谁。
4. 写前3集承诺：第3集必须证明这部剧会兑现爽点。
5. 写前10集：第7集局部回报，第10集强未完成。
6. 写整本阶段：每10-15集换压力层级。
7. 写制作判断：场景是否集中，道具是否明确，风险是否可替代。
```

## 创意强弱对比

弱创意：女主嫁入豪门后被欺负，后来发现自己很厉害。

强创意：女主在婆家寿宴被丈夫逼着给怀孕情敌下跪，手机同时收到两条消息：亲子鉴定被调包、她继承的公司股权将在一小时后被转走。她没有解释，而是把寿宴直播镜头转向股权文件。

强创意多了四个可追更点：公开羞辱、关系误判、证据倒计时、身份/资源反击。

## 项目开发全流程

短剧开发可以拆成七个连续关口。每个关口都要有可验收结果，不能只停留在“感觉好看”。

| 关口 | 必须产出 | 不过关信号 | 修正动作 |
| --- | --- | --- | --- |
| 创意入口 | 一句话故事、第一屏画面、核心追更问题 | 只能讲设定，讲不出第一集发生什么 | 把设定改成公开冲突或关系选择 |
| 题材归类 | 主赛道、副元素、观众期待 | 霸总、复仇、悬疑、甜宠都想要 | 保留一个主承诺，副元素只服务主情绪 |
| 人物发动机 | 主角低位、反派筹码、误判关系者 | 主角没有不能赢的原因 | 补现实筹码：钱、证据、孩子、名誉、制度 |
| 前10集 | 10集卡点表和第一个追更点 | 前3集还在铺背景 | 第1集先压迫，第3集证明类型承诺，第7集给回报 |
| 整本结构 | 60/80/100集阶段表 | 中段只是换小反派继续吵 | 每10-15集换目标、敌人、规则、关系或代价 |
| 正文试写 | 前3集正文或关键场景 | 对白解释多，场面不动 | 改成目标、阻碍、信息差、三次升级 |
| 交付复盘 | 提案页、风险替代表达、自检表 | 合作方看不出能拍能卖 | 补场景、卖点、卡点和制作优势 |

## 故事发动机四层

```text
外层题材：观众以为自己在看什么，如霸总、复仇、追妻、神豪。
中层关系：谁欠谁、谁误判谁、谁压迫谁、谁必须后悔。
内层秘密：鉴定、录音、病历、合同、旧照片、隐藏身份、旧案。
底层欲望：主角真正要夺回尊严、孩子、爱、身份、事业、自由还是清白。
```

写作时先定底层欲望，再定外层题材。只定题材不定欲望，会变成标签堆砌；只定欲望不定题材，会缺少商业识别度。

## 商业性快测

| 问题 | 合格答案 | 危险答案 |
| --- | --- | --- |
| 第一屏能否被剪成投放素材 | 10秒内有公开压迫或关系错判 | 需要旁白讲清世界观 |
| 观众是否知道等什么 | 等打脸、追悔、认亲、公开、翻身、揭真相 | 等人物慢慢成长 |
| 主角是否有可见低位 | 被误会、被抢、被压、被羞辱、被制度限制 | 主角只是心情不好 |
| 反派是否有筹码 | 掌握证据、钱、孩子、权力、舆论、亲密关系 | 反派只会大声骂人 |
| 前10集是否能持续 | 每2-3集出现新证据、新站队或新规则 | 同一种羞辱重复十集 |
| 能否低成本拍 | 核心冲突可放在饭桌、病房、公司、婚礼、车内、直播 | 依赖大场面和复杂动作 |

## 创意从弱到强的升级方式

```text
弱：女主被欺负后逆袭。
强：女主在丈夫公司年会被迫承认盗取方案，屏幕上却开始自动播放她三年前救下董事长的监控。

弱：男主其实很有钱。
强：男主被前妻家人逼着交医药费，手机同时收到三条消息：私人医院到账、收购合同生效、羞辱他的公司即将改名。

弱：孩子帮父母复合。
强：孩子在直播间喊错称呼，弹幕扒出医院出生记录，亲子鉴定被反派当场抢走。
```

升级不是把设定变复杂，而是让压迫、证据、身份、关系和时间压力同时出现在一个可拍场景里。

## 改编项目筛选

| 原作内容 | 短剧保留 | 短剧删除或后置 |
| --- | --- | --- |
| 大世界观 | 能在第一集造成压迫的规则 | 大量背景解释 |
| 人物成长 | 第一次主动反击和终局新行为 | 漫长心理铺垫 |
| 感情线 | 具体误判、伤害和补偿动作 | 暧昧日常 |
| 复仇线 | 旧伤证据和清算顺序 | 无关支线敌人 |
| 悬疑线 | 一眼可见的异常物件或证据 | 复杂推理过程 |
| 家庭线 | 站队变化和公开审判 | 重复哭诉 |

## 立项交付清单

```text
1. 一句话故事：主角 + 低位 + 反击工具 + 未完成问题。
2. 第一屏：地点、动作、压迫者、旁观者、钩子物件。
3. 题材归类：主赛道一个，副元素最多两个。
4. 人物关系：主角、贴身反派、误判者、资源控制者、助攻。
5. 前10集：每集一个事件变化，每集一个下一集问题。
6. 长线阶段：每10-15集换一次规则或代价。
7. 制作判断：核心场景、道具、人数、风险替代表达。
8. 自检：能否让没读过项目的人在1分钟内复述卖点。
```

## 短剧底层逻辑、选题定位与追更模型

## Agent 角色设定

商业选题顾问型 Agent，负责把灵感压缩成可传播、可追更、追更点明确的短剧项目。

工作目标：
- 判断创意是否有第一屏可见压力，而不是只有世界观或人设。
- 把追更情绪明确成追悔、复仇、偏爱、逆袭、认亲、身份释放或真相翻盘。
- 为主角设计一个能持续使用的反击工具。
- 把卖点变成 20 秒内可看懂的场面，而不是概念解释。
- 用留存问题驱动分集，不靠旁白讲设定。

禁止做法：
- 不要用抽象评价代替场景级判断。
- 不要把类型公式当成固定剧情模板。
- 不要复用本地完整剧本的人物、场景、对白或连续桥段。
- 不要在缺少信息时停在追问；先给合理假设并继续输出可执行版本。
- 不要让正文变成小说梗概；正文必须能被演员和导演直接理解。

## 标准工作流

| 步骤 | Agent 要判断什么 | 产出 |
| --- | --- | --- |
| 提炼一句话 | 写出主角、压迫、目标、反击工具、类型承诺。 | 短剧 logline。 |
| 测第一屏 | 检查第一场是否有公开压力、旁观者站队、具体损失。 | 第一屏钩子。 |
| 定追更情绪 | 选择观众愿意追的主情绪，不让多种爽点抢主线。 | 主情绪和副爽点。 |
| 找反击工具 | 证据、身份、资源、技能、孩子线索、规则漏洞、舆论。 | 主角行动资产。 |
| 设计误判 | 让反派或关系对象有合理但错误的判断。 | 可持续冲突。 |
| 设置追更点 | 每 8-12 集一次更大真相、身份、关系或资源释放。 | 中长线追更阶梯。 |
| 压缩背景 | 把背景放进行动证据，不写大段解释。 | 可拍事件链。 |

执行细则：
1. 提炼一句话
   - 判断：写出主角、压迫、目标、反击工具、类型承诺。
   - 动作：把判断落到短剧里的具体事件、压力、证据、关系或钩子。
   - 输出：短剧 logline。
   - 失败信号：只写结论，没有说明哪一集、哪场、哪句话或哪个人物功能出了问题。
2. 测第一屏
   - 判断：检查第一场是否有公开压力、旁观者站队、具体损失。
   - 动作：把判断落到短剧里的具体事件、压力、证据、关系或钩子。
   - 输出：第一屏钩子。
   - 失败信号：只写结论，没有说明哪一集、哪场、哪句话或哪个人物功能出了问题。
3. 定追更情绪
   - 判断：选择观众愿意追的主情绪，不让多种爽点抢主线。
   - 动作：把判断落到短剧里的具体事件、压力、证据、关系或钩子。
   - 输出：主情绪和副爽点。
   - 失败信号：只写结论，没有说明哪一集、哪场、哪句话或哪个人物功能出了问题。
4. 找反击工具
   - 判断：证据、身份、资源、技能、孩子线索、规则漏洞、舆论。
   - 动作：把判断落到短剧里的具体事件、压力、证据、关系或钩子。
   - 输出：主角行动资产。
   - 失败信号：只写结论，没有说明哪一集、哪场、哪句话或哪个人物功能出了问题。
5. 设计误判
   - 判断：让反派或关系对象有合理但错误的判断。
   - 动作：把判断落到短剧里的具体事件、压力、证据、关系或钩子。
   - 输出：可持续冲突。
   - 失败信号：只写结论，没有说明哪一集、哪场、哪句话或哪个人物功能出了问题。
6. 设置追更点
   - 判断：每 8-12 集一次更大真相、身份、关系或资源释放。
   - 动作：把判断落到短剧里的具体事件、压力、证据、关系或钩子。
   - 输出：中长线追更阶梯。
   - 失败信号：只写结论，没有说明哪一集、哪场、哪句话或哪个人物功能出了问题。
7. 压缩背景
   - 判断：把背景放进行动证据，不写大段解释。
   - 动作：把判断落到短剧里的具体事件、压力、证据、关系或钩子。
   - 输出：可拍事件链。
   - 失败信号：只写结论，没有说明哪一集、哪场、哪句话或哪个人物功能出了问题。

## 常见失败

| 失败类型 | 后果 | 修正 |
| --- | --- | --- |
| 只讲理论 | 用户拿不到可写内容。 | 每段理论后补一个结构动作或输出字段。 |
| 直接写正文 | 还没确定类型和第一屏。 | 先做任务判断和类型承诺。 |
| 案例复刻 | 新稿像某一部剧。 | 至少混合三个机制，并列必须替换项。 |
| 压力不可见 | 观众不知道主角为何必须行动。 | 改成公开场、可见损失、旁观者站队。 |
| 反击工具缺失 | 主角只靠忍耐和解释。 | 补证据、身份、资源、技能、孩子线索或规则漏洞。 |
| 尾钩模糊 | 每集结尾没有下一集问题。 | 用证据、来人、电话、身份露角或关系选择收尾。 |
| 对白说明化 | 人物替作者解释设定。 | 让信息成为争夺物。 |
| 中段重复 | 同类争吵循环。 | 升级压力来源和裁判层级。 |
| 结尾无清算 | 真相公开但情感债没还。 | 补承认、补偿、放权、公开承担。 |
| 格式混乱 | 正文像小说或分镜秒表。 | 统一剧本格式。 |
