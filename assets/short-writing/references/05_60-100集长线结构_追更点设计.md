# 60-100 集长线结构与追更点设计

## 1. 长线规划原则

短剧长线要像连续换压力层级，不能像同一件事重复吵。每个阶段都要让人物所处的规则变难。

```text
阶段1：个人羞辱和关系误判。
阶段2：小反派升级为资源封锁。
阶段3：证据线出现，主角开始设局。
阶段4：核心关系对象开始怀疑，但误会更深。
阶段5：中层权力出现，主角小胜变成更大危险。
阶段6：终局秘密逼近公开，所有人重新站队。
阶段7：公开审判和情绪兑现。
```

## 2. 60集、80集、100集差异

| 集数 | 适合项目 | 结构重点 |
| --- | --- | --- |
| 60集 | 引擎单纯、关系集中、成本较低 | 前10集强，20-45集快速升级，终局不拖 |
| 80集 | 常规商业短剧 | 7个阶段，每阶段10-12集，留足中段反转 |
| 100集 | 多秘密、多反派、多关系 | 必须有两次规则改变，否则中段会散 |

## 3. 中段防塌方法

- 换目标：从证明清白，变成保护证人或夺回资源。
- 换敌人：小反派被打脸后，背后资源控制者出场。
- 换关系：误判者开始怀疑，但因为新证据再次站错队。
- 换规则：私人争斗变成公司、法律、舆论或家族规则。
- 换代价：主角继续反击会伤害自己想保护的人。

## 追更点设计原则

```text
追更点不是普通悬念，而是情绪即将兑现时被切断。
弱追更：他到底是谁。
强追更：所有人都等着女主跪下道歉，董事长却拿着亲子鉴定走进来。
弱追更：男主后悔了。
强追更：男主终于找到当年病历，却发现签字放弃抢救的人是自己母亲。
```


## 60-100集长线结构、追更点与中后段升级

## Agent 角色设定

长线结构统筹型 Agent，负责让短剧过了前10集仍有升级、追更点和终局清算。

工作目标：
- 把 60-100 集拆成多个追更段，而不是前10集后重复吵架。
- 为每个段落定义新压力、新证据、新关系状态和新追更问题。
- 让中段从个人误会升级到家族、公司、舆论、法律或权力系统。
- 让结尾清算覆盖情感、名誉、资源、关系和身份。
- 避免第30集以后靠拖延误会维持长度。

禁止做法：
- 不要用抽象评价代替场景级判断。
- 不要把类型公式当成固定剧情模板。
- 不要复用本地完整剧本的人物、场景、对白或连续桥段。
- 不要在缺少信息时停在追问；先给合理假设并继续输出可执行版本。
- 不要让正文变成小说梗概；正文必须能被演员和导演直接理解。

   - 失败信号：只写结论，没有说明哪一集、哪场、哪句话或哪个人物功能出了问题。

## 诊断表

| 问题 | 表面症状 | 对应修复动作 |
| --- | --- | --- |
| 中段重复 | 同一误会拖几十集。 | 每段换压力来源或证据状态。 |
| 追更点稀薄 | 长时间无真相、身份或关系变化。 | 每 8-12 集安排一次强承诺兑现。 |
| 反派断层 | 小反派失败后没有新对手。 | 增加后台、制度、家族或公司裁判。 |
| 主角工具不升级 | 一直用同一证据打脸。 | 让工具从小证据升级到资源、身份、规则。 |
| 关系线停住 | 追妻、偏爱或认亲没有状态变化。 | 每段改变关系选择或信任程度。 |
| 结尾清算太快 | 最后两集突然公开真相。 | 提前埋终局证据和公开场合。 |
| 副线抢主线 | 中段开太多新人物。 | 副线必须服务主追更情绪。 |
| 反转无代价 | 每次反转只是信息变化。 | 反转必须造成损失、站队或行动改变。 |

逐项诊断写法：
### 诊断 1：中段重复

- 诊断信号：同一误会拖几十集。
- 深层原因：短剧的压力、行动、证据、关系或钩子没有承担应有功能。
- 修复动作：每段换压力来源或证据状态。
- 输出要求：指出原材料中 1-3 个具体位置，再给出可直接替换的结构或场景写法。
- 自检问题：修复后观众是否能更快理解压力、更想等下一集、且不需要旁白解释？

### 诊断 2：追更点稀薄

## 常见失败

| 失败类型 | 后果 | 修正 |
| --- | --- | --- |
| 只讲理论 | 用户拿不到可写内容。 | 每段理论后补一个结构动作或输出字段。 |
| 直接写正文 | 还没确定类型和第一屏。 | 先做任务判断和类型承诺。 |
| 案例复刻 | 新稿像某一部剧。 | 至少混合三个机制，并列必须替换项。 |
| 压力不可见 | 观众不知道主角为何必须行动。 | 改成公开场、可见损失、旁观者站队。 |
| 反击工具缺失 | 主角只靠忍耐和解释。 | 补证据、身份、资源、技能、孩子线索或规则漏洞。 |
| 尾钩模糊 | 每集结尾没有下一集问题。 | 用证据、来人、电话、身份露角或关系选择收尾。 |
| 对白说明化 | 人物替作者解释设定。 | 让信息成为争夺物。 |
| 中段重复 | 同类争吵循环。 | 升级压力来源和裁判层级。 |
| 结尾无清算 | 真相公开但情感债没还。 | 补承认、补偿、放权、公开承担。 |
| 格式混乱 | 正文像小说或分镜秒表。 | 统一剧本格式。 |

## 自检清单

- [ ] 是否先判断了用户任务，而不是直接写？
- [ ] 是否明确了主类型和主追更情绪？
- [ ] 第一屏是否有公开压力、可见损失和旁观者站队？
- [ ] 主角是否有可重复使用的反击工具？
- [ ] 反派或压迫者是否掌握具体资源、规则或证据？
- [ ] 每一集是否有事件，而不是只有情绪对话？
- [ ] 每一集结尾是否留下具体下一集问题？
- [ ] 前3集是否完成压迫、误判和小反击？
- [ ] 前10集是否打开中段更大压力？
- [ ] 60-100集是否有分段追更点？
- [ ] 人物关系是否有可拍的关系债？
- [ ] 对白是否有目标、地位差、信息差和转折？
- [ ] 网文改编是否删除了慢铺垫和无效支线？
- [ ] 对标案例是否只借机制，不借原文表达？
- [ ] 是否列出必须替换的人物、证据、职业、场景和反转顺序？
- [ ] 正文是否使用第几集/场次/场景/人物/动作/对白格式？
- [ ] 质量自检是否指出了仍需补强的地方？

## 操作卡片

### 1-10集段

用途：让观众进入主情绪。

必须检查：
- 类型承诺
- 压迫
- 工具启动
- 第10集开门

输出方式：
```text
【1-10集段】
判断：
可用机制：
需要替换：
落到本项目的写法：
```

### 11-25集段

用途：从个人冲突升到资源冲突。

必须检查：
- 小清算
- 关系试探
- 后台露角

输出方式：
```text
【11-25集段】
判断：
可用机制：
需要替换：
落到本项目的写法：
```

### 26-40集段

用途：让主角旧方法失效。

必须检查：
- 伪胜利
- 反噬
- 更高裁判

输出方式：
```text
【26-40集段】
判断：
可用机制：
需要替换：
落到本项目的写法：
```

### 41-60集段

用途：把局面推到不可回头。

必须检查：
- 系统压力
- 身份/证据逼近公开
- 关系站队

输出方式：
```text
【41-60集段】
判断：
可用机制：
需要替换：
落到本项目的写法：
```

### 终局段

用途：清算必须覆盖开场旧伤。

必须检查：
- 公开场合
- 全证据链
- 行动补偿
- 新选择

