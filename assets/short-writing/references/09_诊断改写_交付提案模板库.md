# 诊断改写、交付提案与模板库

## 诊断、改写与评分

## 1. 诊断流程

先判断问题属于选题、类型、人物、前10集、长线、场景、对白、钩子、交付哪一类，再给优先级。诊断必须指出缺失机制、具体场景证据和可执行修复动作。

## 2. 评分表

| 维度 | 满分 | 扣分信号 |
| --- | --- | --- |
| 第一屏 | 10 | 开头解释背景、没有即时冲突 |
| 题材承诺 | 10 | 霸总不像霸总，复仇没有清算，追妻没有追悔 |
| 主角压迫位 | 10 | 主角没有可见低位或不能赢的原因 |
| 反击工具 | 10 | 主角只能忍，没有证据、身份、能力或盟友 |
| 前10集 | 15 | 重复羞辱，没有局部回报和强未完成 |
| 长线 | 15 | 中段不换规则，只换小反派 |
| 场景 | 10 | 目标、阻碍、信息差、转折不足 |
| 对白 | 10 | 解释多，行动少，人物同声 |
| 钩子 | 10 | 下一集问题不明确 |

## 交付规范与提案方法

## 1. 交付优先级

短剧项目交付要让制片、平台或合作方快速判断：能不能拍、能不能卖、前10集强不强、风险能不能规避。

## 2. 提案包结构

```text

一句话故事：
题材类型：
目标观众：
核心卖点：
第一集开场：
前10集卡点：
80集/100集阶段表：
人物关系图：
制作优势：
边界风险与替代表达：
```

## 3. 风险替代表达

- 极端暴力改为结果展示、证据展示或旁人反应。
- 违法复仇改为公开证据、法律程序、舆论压力或资源制裁。
- 低俗擦边改为关系压力、误会、羞辱和选择困境。
- 封建迷信改为人物相信、骗局、心理恐惧或类型化悬疑。

## 4. 交付自检

前10集是否能独立评估；每集是否有钩子；场景是否可拍；角色功能是否清楚；风险点是否有替代方案。

## 诊断、改写、网文改编与交付模板

## Agent 角色设定

短剧医生与交付总编型 Agent，负责先诊断问题，再给出可执行改写和完整交付包。

工作目标：
- 任何改写前先判断问题属于选题、类型、人物、结构、场景、对白还是原创化风险。
- 网文改编必须删慢铺垫、合并支线、前置第一屏压力。
- 用户给完整文件时，先做章节摘要、人物关系表和冲突资产表。
- 交付模板要能直接进入创作，不只写建议。
- 每次正文输出都带质量自检和下一步可扩写方向。

禁止做法：
- 不要用抽象评价代替场景级判断。
- 不要把类型公式当成固定剧情模板。
- 不要复用本地完整剧本的人物、场景、对白或连续桥段。
- 不要在缺少信息时停在追问；先给合理假设并继续输出可执行版本。
- 不要让正文变成小说梗概；正文必须能被演员和导演直接理解。

## 标准工作流

| 步骤 | Agent 要判断什么 | 产出 |
| --- | --- | --- |
| 问题归类 | 判断是结构、人物、类型、场景、对白、改编还是风险。 | 诊断类型。 |
| 找证据 | 从用户材料中指出具体慢点、缺点或可改节点。 | 问题证据。 |
| 定策略 | 决定是前置、删除、合并、转场、升级、换工具还是重写。 | 改写策略。 |
| 给模板 | 按项目包、前10集、正文、诊断报告或改编方案输出。 | 交付结构。 |
| 写正文 | 需要正文时按标准剧本格式写。 | 可拍文本。 |
| 做自检 | 检查第一屏、每集事件、钩子、原创化和格式。 | 质量报告。 |
| 给下一步 | 指出最应该继续补的材料。 | 后续任务。 |

执行细则：
1. 问题归类
   - 判断：判断是结构、人物、类型、场景、对白、改编还是风险。
   - 动作：把判断落到短剧里的具体事件、压力、证据、关系或钩子。
   - 输出：诊断类型。
   - 失败信号：只写结论，没有说明哪一集、哪场、哪句话或哪个人物功能出了问题。
2. 找证据
   - 判断：从用户材料中指出具体慢点、缺点或可改节点。
   - 动作：把判断落到短剧里的具体事件、压力、证据、关系或钩子。
   - 输出：问题证据。
   - 失败信号：只写结论，没有说明哪一集、哪场、哪句话或哪个人物功能出了问题。
3. 定策略
   - 判断：决定是前置、删除、合并、转场、升级、换工具还是重写。
   - 动作：把判断落到短剧里的具体事件、压力、证据、关系或钩子。
   - 输出：改写策略。
   - 失败信号：只写结论，没有说明哪一集、哪场、哪句话或哪个人物功能出了问题。
4. 给模板
   - 判断：按项目包、前10集、正文、诊断报告或改编方案输出。
   - 动作：把判断落到短剧里的具体事件、压力、证据、关系或钩子。
   - 输出：交付结构。
   - 失败信号：只写结论，没有说明哪一集、哪场、哪句话或哪个人物功能出了问题。
5. 写正文
   - 判断：需要正文时按标准剧本格式写。
   - 动作：把判断落到短剧里的具体事件、压力、证据、关系或钩子。
   - 输出：可拍文本。
   - 失败信号：只写结论，没有说明哪一集、哪场、哪句话或哪个人物功能出了问题。
6. 做自检
   - 判断：检查第一屏、每集事件、钩子、原创化和格式。
   - 动作：把判断落到短剧里的具体事件、压力、证据、关系或钩子。
   - 输出：质量报告。
   - 失败信号：只写结论，没有说明哪一集、哪场、哪句话或哪个人物功能出了问题。
7. 给下一步
   - 判断：指出最应该继续补的材料。
   - 动作：把判断落到短剧里的具体事件、压力、证据、关系或钩子。
   - 输出：后续任务。
   - 失败信号：只写结论，没有说明哪一集、哪场、哪句话或哪个人物功能出了问题。

### 诊断 1：诊断泛化

- 诊断信号：只给结论不指材料。
- 深层原因：短剧的压力、行动、证据、关系或钩子没有承担应有功能。
- 修复动作：每条问题绑定用户文本中的具体情节。
- 输出要求：指出原材料中 1-3 个具体位置，再给出可直接替换的结构或场景写法。
- 自检问题：修复后观众是否能更快理解压力、更想等下一集、且不需要旁白解释？

- 输出要求：指出原材料中 1-3 个具体位置，再给出可直接替换的结构或场景写法。

- 自检问题：修复后观众是否能更快理解压力、更想等下一集、且不需要旁白解释？

### 诊断 8：对标风险

- 诊断信号：改写后仍像原剧。
- 深层原因：短剧的压力、行动、证据、关系或钩子没有承担应有功能。
- 修复动作：换关系债、证据物件、职业场景和反转顺序。
- 输出要求：指出原材料中 1-3 个具体位置，再给出可直接替换的结构或场景写法。
- 自检问题：修复后观众是否能更快理解压力、更想等下一集、且不需要旁白解释？

## 常见失败

| 失败类型 | 后果 | 修正 |
| --- | --- | --- |
| 只讲理论 | 用户拿不到可写内容。 | 每段理论后补一个结构动作或输出字段。 |
| 直接写正文 | 还没确定类型和第一屏。 | 先做任务判断和类型承诺。 |
| 案例复刻 | 新稿像某一部剧。 | 至少混合三个机制，并列必须替换项。 |
| 压力不可见 | 观众不知道主角为何必须行动。 | 改成公开场、可见损失、旁观者站队。 |
| 反击工具缺失 | 主角只靠忍耐和解释。 | 补证据、身份、资源、技能、孩子线索或规则漏洞。 |
| 尾钩模糊 | 每集结尾没有下一集问题。 | 用证据、来人、电话、身份露角或关系选择收尾。 |
| 对白说明化 | 人物替作者解释设定。 | 让信息成为争夺物。 |
| 中段重复 | 同类争吵循环。 | 升级压力来源和裁判层级。 |
| 结尾无清算 | 真相公开但情感债没还。 | 补承认、补偿、放权、公开承担。 |
| 格式混乱 | 正文像小说或分镜秒表。 | 统一剧本格式。 |

## 自检清单

- [ ] 是否先判断了用户任务，而不是直接写？
- [ ] 是否明确了主类型和主追更情绪？
- [ ] 第一屏是否有公开压力、可见损失和旁观者站队？
- [ ] 主角是否有可重复使用的反击工具？
- [ ] 反派或压迫者是否掌握具体资源、规则或证据？
- [ ] 每一集是否有事件，而不是只有情绪对话？
- [ ] 每一集结尾是否留下具体下一集问题？
- [ ] 前3集是否完成压迫、误判和小反击？
- [ ] 前10集是否打开中段更大压力？
- [ ] 60-100集是否有分段追更点？
- [ ] 人物关系是否有可拍的关系债？
- [ ] 对白是否有目标、地位差、信息差和转折？
- [ ] 网文改编是否删除了慢铺垫和无效支线？
- [ ] 对标案例是否只借机制，不借原文表达？
- [ ] 是否列出必须替换的人物、证据、职业、场景和反转顺序？
- [ ] 正文是否使用第几集/场次/场景/人物/动作/对白格式？
- [ ] 质量自检是否指出了仍需补强的地方？
