# Phase 1：剧本创作

> Phase 1 核心任务：**从一句话创意到12项可交付的剧本项目包**。调用本技能的内置创作资产（9篇规则参考 + 875条蒸馏案例 + 321部加密剧本）。

## 1.0 创作工作流（6步法）

```
一句话创意
  │
  ├─ 第1步 定位赛道 ─────────── 读 references/02_类型矩阵，搜蒸馏库验证
  │  产出：【主赛道与副元素】+【核心卖点一句话】
  │
  ├─ 第2步 设计人物 ─────────── 读 references/03_人物关系，搜蒸馏库找压迫模型
  │  产出：【人物关系与压迫系统】
  │
  ├─ 第3步 打造第一屏 ──────── 读 references/04_前10集卡点，搜蒸馏库找钩子
  │  产出：【第一屏压力】+【目标受众与追更情绪】
  │
  ├─ 第4步 规划全剧 ────────── 读 references/05_长线结构，搜蒸馏库找升级节点
  │  产出：【前10集分集表】+【60-100集长线结构】
  │
  ├─ 第5步 写正文 ──────────── 读 references/06_场景对白，参照蒸馏库节拍模式
  │  产出：【前5集正文】
  │
  └─ 第6步 自检 ────────────── 对照 references/09_诊断改写，查漏补缺
     产出：【任务判断】+【调用资料与案例机制】+【原创化自检】+【质量自检】
```

### 1.1.2 赛道定位三步

1. **选主赛道**：从速查表中选一个最匹配创意的频道×赛道
2. **加副元素**：从其他赛道借1-2个爽点元素作为调味（如霸总+萌宝=带球跑的总裁文）
3. **验证可行性**：`search_library.py "<赛道关键词> 第一屏" --top 5`，确认该赛道有足够的爆款案例支撑

> 路径速记：`SK=~/.hermes/skills/短剧大师/assets/short-writing`

## 1.2 第二步：设计人物（角色构建）

### 1.2.1 角色功能位（每剧必备5类角色）

| 功能位 | 定义 | 设计要求 | 示例方向 |
|:--:|------|------|------|
| **主角** | 承载观众代入感的核心 | 有隐藏优势但处于低位，有能力但被低估 | 隐藏身份的强者/被误判的普通人 |
| **女主** | 三步阶梯：互利→守护→并肩 | 从功利关系进入，逐步转化为情感羁绊 | 最初为利益接近，最后为情感留下 |
| **第一反派** | 前10集的主要压迫来源 | 具体可感的短期威胁，被击败后掉落遗产 | 地方恶霸/觊觎者/竞争者 |
| **终极反派** | 60集长线的最终目标 | 在前期只暗示存在，中期逐步揭示 | CEO/幕后黑手/更高层势力 |
| **成长伙伴** | 主角在本剧成长的见证者 | 有独立弧线，不是工具人 | 最初被胁迫/被抛弃→最终选择站队主角 |

### 1.2.2 人物关系压迫系统公式

```
上位压迫者：拥有主角需要的资源/身份/认可
      ↓ 施加
可见压迫：公开羞辱 / 资源拒绝 / 身份否定 / 情感误判
      ↓ 触发
主角反应：不解释 / 局部反击 / 证据留存
      ↓ 升级
更大压迫：上一级压迫者介入 / 旧账翻出 / 规则改变
```

> 每2-3集一个压迫→反击→升级的循环，前10集约3-4个循环。

### 1.2.3 人物构建自检

- ✅ 主角有一个明确且可视觉呈现的「隐藏身份/能力」
- ✅ 主角的弱点/缺陷在第一屏就可见（让观众有代入感）
- ✅ 反派的压迫有具体场景（饭桌/会议/审判），不是抽象威胁
- ✅ 每个主要角色有一个「观众能记住的标签」——外貌/口头禅/习惯动作
- 🔴 主角不能全知全能——金手指必须有代价（见§2.3.6）

## 1.3 第三步：打造第一屏（前10集卡点）

### 1.3.1 第一屏公式

```
E01: 公开羞辱 + 金手指信号（画面暗示，不直说）
E02: 羞辱后果 + 第一个小证据
E03: 局部反击 + 更大压迫出现
E04-E06: 第二轮压迫→反击→升级
E07-E09: 第三轮，反派从个人升级到组织/家族
E10: 第一反派落幕 + 掉落遗产 + 尾钩揭示「背后还有更大的」
```

### 1.3.2 钩子密度要求

| 指标 | 标准 |
|------|------|
| 每集至少1个信息揭露 | "原来/其实/竟是" |
| 每2-3集一次身份能力释放 | 逐层释放能力/资源/旧部/身份 |
| 前3集必须出现金手指的视觉信号 | 每次起效时有统一视觉符号 |
| E01的尾钩必须有「怎么回事？」级别的悬念 | 见§2.3.2五类尾钩 |

## 1.4 第四步：规划全剧（长线结构）

### 1.4.1 60集五段结构骨架

```
E01-E10  建立期：日常→异常→金手指初显→第一个反派→获得第一批资源
E11-E20  展开期：第二个反派→势力介入→主角开始意识到自己的不同
E21-E35  升级期：幕后势力登场→金手指代价开始显现→主角第一次主动选择
E36-E50  转折期：代价累积→关键配角立场选择→主角面临重大抉择
E51-E60  终局期：最终反派→所有人站队→开放式结局→为续季留钩子
```

### 1.4.2 分集规划模板

| 集段 | 核心事件 | 金手指阶段 | 反派档位 | 暗线状态 |
|------|------|------|:--:|------|
| E01-E05 | 建立压迫+金手指初显 | 无意使用 | 第1档 | A首现 |
| E06-E10 | 第一反派落幕+遗产掉落 | 外界发现 | 第1-2档 | B首现 |
| E11-E20 | 第二反派+势力介入 | 开始被争夺 | 第2-3档 | B/C推进 |
| E21-E30 | 幕后势力亮相 | 首次被窃取 | 第3-4档 | C/D首现 |
| E31-E45 | 代价显现+重大抉择 | 首次主动使用 | 第4-5档 | D加速 |
| E46-E60 | 终局战+开放式季终 | 选择权在主角 | 第5-6档 | 全揭示 |

## 1.5 第五步：写正文（场景与对白）

### 1.5.1 正文格式

```
第N集 / 第M场 / 场景地点 / 出场人物 / 动作描述 / 对白
```

禁止"几秒到几秒"格式（那是Phase 2升级时才做的事）。

### 1.5.2 场景写作原则

| 原则 | 说明 | 反例 |
|------|------|------|
| **一景一事** | 每个场景只推进一个核心事件 | 吃饭时顺便谈生意+揭露身世+表白 |
| **对白不解释** | 用动作和画面传递信息，不对白直述身世 | "其实我是XX家族的继承人" |
| **冲突前置** | 每场开头3句内出现矛盾信号 | 寒暄3分钟才进入正题 |
| **尾钩回收** | 每集末场为下一集留下必须看的理由 | 本集圆满收尾 |
| **信息渐进** | 每集新信息块≤7个（Miller 7±2） | 一集塞进5个新角色+3个新规则 |

### 1.5.3 对白写作铁律

- ✅ 单句≤25字（偶尔30字可过）
- ✅ 同一角色固定1个称呼（不能一会叫"总裁"一会叫"傅总"）
- ✅ 保留反讽/双关/潜台词（口语化不等于低智化）
- ❌ 书面语/文言文
- ❌ 无解释的修炼/阵法/族名术语
- ❌ 外语/英文缩写

### 1.5.4 仿写边界

从蒸馏库和加密剧本库中提取的是**机制级规律**：压力逻辑、钩子密度、反转节奏、信息揭露的时机。不复刻：人物姓名、具体对白、完整场景、连续剧情。

## 1.6 第六步：自检（质量审查）

### 1.6.1 原创化自检清单

- [ ] 主角的功能位与任何对标剧不同（换身份/换背景/换困境）
- [ ] 对白无逐句复刻任何参考剧
- [ ] 场景设置不重复对标剧的连续三个场景
- [ ] 压力来源和爽点释放方式有独立的逻辑链

### 1.6.2 质量自检清单

- [ ] 前3集出现金手指视觉信号
- [ ] E01尾钩有「怎么回事？」级别悬念
- [ ] 主角的隐藏身份/能力在E10前至少释放1层
- [ ] 第一反派在E10前落幕并掉落遗产
- [ ] 女主入场动机清晰（互利→守护→并肩的第一步）
- [ ] 每集至少1个信息揭露

## 1.7 交付物格式（12项·逐项说明）

```markdown
【任务判断】
  一句话总结本次创作的核心任务和边界。

【调用资料与案例机制】
  列出创作中调用了哪些references文档、哪些蒸馏案例卡（含ID）、哪些加密剧本。
  格式：参考 02_类型矩阵 + 案例卡 c0005 c0012 / 加密剧本「剧名」

【主赛道与副元素】
  主赛道（必选）× 副元素（1-2个调味）。
  例：主赛道=霸总 × 副元素=萌宝+复仇

【核心卖点一句话】
  用「表面X × 隐藏Y」格式。
  例：表面被逐出家门的弃少 × 隐藏身价百亿的财阀继承人

【目标受众与追更情绪】
  目标受众画像 + 他们「为什么必须追下一集」？
  例：25-35岁女性 / 追更情绪：想知道前夫发现真相后的反应

【第一屏压力】
  前3集的具体压迫事件序列（每集一句话描述压迫→反击→升级的循环）

【人物关系与压迫系统】
  5个核心角色的功能位、压迫关系、独立弧线。
  格式见 §1.2.1 角色功能位 + §1.2.2 压迫系统公式

【前10集分集表】
  E01-E10 每集一句话的核心事件。
  格式：E01:__ / E02:__ / ... / E10:__

【60-100集长线结构】
  按 §1.4.1 五段骨架，每段2-3句话的核心推进方向

【前5集正文】
  E01-E05 完整正文，格式见 §1.5.1

【原创化自检】
  按 §1.6.1 清单逐项打勾

【质量自检】
  按 §1.6.2 清单逐项打勾
```

# Phase 2：剧本升级

> Phase 1 产出标准短剧项目包（含12项交付物）后，**取其"前5集正文"和"60-100集长线结构"为基础**，进入 Phase 2 升级。
> Phase 2 的核心转换：**叙事文本 → 可量化剧本**。把"什么发生了"的叙事正文，转换为"每集几镜/每镜几秒/爆点在哪"的标准化格式。
> 全季均有效——新项目从零启动，无一集豁免。

## 2.0 Phase 1→Phase 2 衔接规则

| Phase 1 产出 | Phase 2 输入 | 转换方式 |
|------|------|------|
| 【前5集正文】 | 集级5/6镜宏镜头版 | 每篇正文→每集N镜+内部节拍+对白+旁白 |
| 【60-100集长线结构】 | 暗线A/B/C/D标注 | 长线事件→暗线状态值+首现集+揭示集 |
| 【人物关系与压迫系统】 | 武打L1/L2/L3触发定位 | 人物冲突点→武打效果分级 |

## 2.2 暗线标注框架（通用）

> 暗线 = 贯穿全剧的伏笔线索，每集至少出现1次，进度单调前进不可倒退。以下为通用模板框架，按项目实际大纲填充。

| 暗线 | 名称 | 主线内容（填项目内容） | 首现集 | 揭示集 | 覆盖要求 |
|:---:|------|---------|-------|--------|---------|
| **A** | 主线物品/力量 | 从沉睡到苏醒·从隐到显→枯竭/闭合/消失 | 项目定 | 项目定 | 每集≥1处视觉提示 |
| **B** | 封印/限制 | 从完整→碎裂→修复/失效 | 项目定 | 项目定 | 首现起≥1镜/集 |
| **C** | 深层威胁 | 已知问题的背后有更大存在 | 项目定 | 项目定 | 暗示·不露面·后期逐步明确 |
| **D** | 代价/牺牲 | 主角/重要角色被动渗透→觉醒→主动选择 | 项目定 | 项目定 | 每集≥1镜·监控关键状态 |

> **标注规则**：
> - 暗线A标状态值（如：充沛/波动/消耗/枯竭）
> - 暗线B标完整度（如：7/7→6/7→…→0/7→重新闭合）
> - 暗线C标威胁等级（如：沉睡/异动/苏醒暗示/明确威胁）
> - 暗线D标代价进度（如：无感/偶尔症状/明显代价/临界）
> - 所有进度单调不倒退
> - 暗线只能在镜头正文中标注（不得仅出现在SQ/参数表）

## 2.3 爆款节奏版（5/6镜65秒）

> **镜数规则**：标准集=5镜 / 武打高潮集=6镜 / 终局集=6镜。高潮集清单见下。

> **内部节拍** = 宏镜头内的秒级时序分段，格式：`0-2s：{动作/对白/旁白/特效}`。每镜含 2-4 个内部节拍。节拍时长之和 = 该镜时长。所有节拍时长之和 = 全片声明时长。

| 指标 | 标准 | 检测方法 |
|------|------|---------|
| **镜数** | 标准集=5 / 高潮集=6 / 终局集=6 | 计数 |
| **时长** | 标准集60-65s / 高潮集65-70s / 终局集=70s | 节拍求和=声明时长±1s |
| **爆点密度** | ≥12爆点/集，最大间隔≤8s | 按内部节拍逐段检测（见SQ8爆点计数法） |
| **爆点定义** | 情绪转折 / 信息揭露 / 武打L2+ / 对白冲突 / 视觉特效 | 五类任一=1爆点 |
| **旁白** | ≤4段，总旁白≤全片20% | 计数+计时 |
| **镜长均衡** | 最短≥8s 最长≤16s 长-短差≤8s（6镜集min≥8s max≤14s range≤6s） | 逐镜计时 |
| **景别** | ≥8次景别变化/集 | 计数 |
| **细节** | ≥8处细节描述/集（材质/光影/表情/环境质感） | 计数 |
| **对白** | 对话密集短促，无长独白 | 人工检查 |
| **冷场** | 单一连续沉默≤2s（情感低点除外·可达3-5s） | 计时 |
| **情感低点** | ≥1处安静/留白≥3秒/集（防止全片扁平化） | 计数 |
| **节奏模板·五幕** | 前情(5s)→冲突(20s)→关系(20s)→小结(14s)→尾钩(6s) | 逐镜计时 |

## 3.3 四种工作流

## 3.4 轮询策略

- **间隔**：10秒/次
- **增量**：首次 after-seq=0，后续根据消息数计算
- **完成判定**：run结束且content含产物URL
- **超时**：连续48h无结果→告知用户
- **重试**：单次失败重试1次，连续3次失败停止

## 3.5 核心原则：用户侧只做搬运工

## SQ4 · 逻辑连贯性审查（A13禁制·L5）


### L5 五维速查

| 编号 | 维度 | 一句话标准 |
|:--:|------|------|
| L1 | 因果完整 | 每件事有「因为→所以」，不断链 |
| L2 | 时空清晰 | 观众知道每镜在哪、什么时候 |
| L3 | 信息有度 | 新信息块≤7个/集（Miller 7±2），不轰炸 |
| L4 | 台词干净 | 口语化，术语配解释，单句≤25字，反讽/双关/潜台词保留 |
| L5 | 情感节奏 | 每集1-2个情感高点，节奏自然不硬塞 |

### 六维检查（附L5映射）

| 维度 | 通过标准 | L5映射 | 模式 |
|------|---------|------|:---:|
| 大纲事件覆盖 | 本集大纲条目在正文中各有对应 | L1因果完整 | 🔧 |
| 时间线连续 | 无倒流、无缺过渡的跳跃（跨度≥半天且有过渡=合法） | L2时空清晰 | 🔧 |
| 逻辑连贯 | 事件因果/角色动机/信息渐进均按项目设定逻辑体系 | L1因果完整 | 🔧 |
| 超前拦截 | 0命中重校准禁词表 | L3信息有度 | 🤖 |
| 剧情推进 | 本集≥1核心事件 | L5情感节奏 | 🔧 |
| 人物一致 | 行为不OOC（OOC=角色言行偏离大纲性格设定·判断依据=本集行为与已有集数建立的角色性格模式一致） | L1因果完整(动机) | 🔧 |

### 五类逻辑断裂（任一命中=A13不通过）

> 五类断裂与L5五维的映射：L1因果→断裂①③④ | L2时空→断裂②⑤ | L3信息→断裂④ | L4台词→(独立检查·无断裂对应) | L5情感→(独立检查·无断裂对应)

1. 因果缺失 2. 空间跳跃 3. 动机空白 4. 信息幽灵 5. 时间坍缩



##### L1-因果完整

每件事的发生能从前面找到「为什么」。允许暗线，但不允许断链。
- 事件A（因）→ 事件B（果），中间不能缺环节
- 角色知道的每件事都必须有获取过程（不能"忽然就知道"）
- 重要决定的动机必须可理解（一句话能说清"他为什么这么做"）

##### L2-时空清晰

转场必须有过渡（动作/声音/光变化）。跨场景必须有建立镜头。
- 空间跳跃→加过渡（走/推门/光变化）+ 新场景建立镜头
- 时间跨度≥1天→标注（"第二天""三日后"）
- 昼夜变化→天色渐变/钟声/点灯

##### L3-信息有度

回归 Miller 7±2。新信息块≤7个/集（含新角色/新概念/新关系/新规则）。
- 新概念首次出现→用一句话白话解释，解释本身不引入更多新术语
- 每镜最多引入2个新信息块
- 术语轰炸=不通过

##### L4-台词干净

口语化，但**保留**反讽/双关/潜台词。

| 允许 | 禁止 |
|------|------|
| 反讽/双关/潜台词 | 书面语/文言文 |
| 单句≤25字（偶尔30字可过） | 无解释的修炼/阵法/族名术语 |
| 同一角色固定1个称呼 | 随机混用称呼 |
| 术语首次出现配白话解释 | 外语/英文缩写 |

##### L5-情感节奏

每集不少于1-2个情感高点。不硬塞——节奏自然比数量更重要。
- 开场镜有情绪信号
- 情感类型有交替（不能全镜紧张或全镜平淡）

**L5.1 情绪三通病（铭兴映画·短剧情绪刻画）**

> 来源：铭兴映画《短剧质感总差点意思？问题卡在情绪刻画上》。大多数短剧质感上不去，不是技术问题，是情绪太直白、太满、太刻意。

| 通病 | 表现 | 检测方法 | 项目红线 |
|------|------|------|------|
| ①情绪过度外放 | 全程大喊大叫、嘶吼争辩、台词满是戾气、无层次起伏 | 扫剧本对白行：连续≥3行含`怒\|吼\|叫\|骂\|哭` → WARN | 按项目设定：主角不能无理由大喊大叫——情绪克制才是高级表达 |
| ②情绪没有铺垫 | 上一秒平静下一秒崩溃，无心理过渡无细节支撑 | 每个情绪高点的前3镜必须有情绪过渡标记（眼神/手/呼吸变化） → 缺失=ERROR | 按项目设定：每个情绪爆发点前应有≥3处伏笔逐级递进 |
| ③只靠台词堆情绪 | 大段对白宣泄，不用眼神/神态/场景烘托，毫无留白 | 情绪高点镜：对白占比≤50%（另一半留给光影/动作/沉默） → 超标=WARN | 按项目设定：最重的话一定最短，情绪靠画面传递而非台词 |

**L5.2 情绪三解法**

| 解法 | 操作 | 检测标准 |
|------|------|------|
| ①克制表达 | 攥紧的拳头 > 嘶吼。泛红的眼眶 > 哭嚎。沉默转身 > 控诉。压抑到极致再适度释放 | 每集≥1处"克制时刻"（情绪即将爆发但被角色主动压住）|
| ②情绪铺垫 | 每个爆发点前有≥3处伏笔。委屈→隐忍→猜忌→积累→释放。线条连贯不跳 | 爆发前伏笔≥3处（可跨集追溯）|
| ③情绪留白 | 不用把心里话全说透。用场景氛围+人物神态+慢镜头烘托。留一部分给观众体会 | 每集≥1处"留白时刻"（角色沉默≥2s + 无内心独白 + 画面传递情绪）|

**L5.3 情绪节奏校验**

```
每集检测：
- 克制时刻 ≥1（情绪外放检测）
- 留白时刻 ≥1（情绪留白检测）
- 情绪高点的前3镜均有过渡标记（情绪铺垫检测）
- 情绪高点镜对白占比 ≤50%
任一不满足 → ERROR
```

##### 五类逻辑断裂 + 修复方法

1. **因果缺失**：事件B在事件A之后，但无因果连接 → 补一句台词/画面解释
2. **空间跳跃**：角色在无过渡的情况下出现在新地点 → 加过渡镜头（走路/推门/光变化）
3. **动机空白**：角色做出重要决定但未展示思考过程 → 加内心独白或表情特写
4. **信息幽灵**：角色知道了他本不该知道的信息 → 加镜头展示获取信息过程
5. **时间坍缩**：时间跨度不合理无过渡 → 加天色/环境变化过渡

### A13禁制

> **五类断裂任一命中 → A13 → 不通过。** 每集逻辑打磨≥5轮。

### 4.5.1 竖屏七法（每集 ≥3 种）

| # | 构图法 | 核心 | 竖屏提示词 | 适用场景示例 |
|:--:|------|------|------|------|
| ① | 中心对称 | 主角中轴居中·配角左右对称 | "center composition, symmetrical framing, centered subject" | 觉醒/涅槃/归心 |
| ② | 纵深分层 | 前景遮挡→中景主角→远景环境 | "depth layering, foreground frame, midground subject, background depth" | 日常/对话/潜行 |
| ③ | 竖帧留白 | 下2/3人物 + 上1/3环境/天空/建筑 | "vertical headroom, negative space upper third, tall frame composition" | 独处/思考/仰望 |
| ④ | 低角仰拍 | 低机位仰拍·角色占满竖屏9:16高度 | "low angle hero shot, figure fills vertical 9:16 frame" | 威慑/登场/权力 |

| ⑥ | 运动跟随 | 跟拍 + 前景快速掠过增强速度感 | "tracking shot, foreground motion blur, speed parallax" | 追逐/战斗/疾走 |
| ⑦ | 微广角 | 近景主体微变形 + 背景压缩·强调压迫感 | "slight wide angle, close subject distortion, compressed background" | 对峙/压迫/情绪爆发 |

> **审查**：每集 ≥3 种不同构图法。宏镜头标注格式：`【竖屏七法·{编号}{名称}】`（如 `【竖屏七法·②纵深分层】`）。七法编号与镜头内容语义须匹配（不可在静态对话镜标⑥运动跟随·在独白镜标④低角仰拍等）。

> **运镜搭配**：竖屏七法解决"主体在画面中怎么摆"（静态构图），具体"镜头怎么动"（运镜方式）见 J.3.2 20种运镜百科。

## J.3 五段式提示词公式（SQ6.3新增）

> 替代长文堆词。每镜按此公式组装prompt。

### 公式

```
【固定主体】{角色名/外观关键词，保持发型/服装/面部特征不变}

【一个核心动作】{单一动作目标，不多写}
+
【固定场景/时间/光线】{场景+时间+光源方向+色调}
+
【一种主运镜】{仅一种：固定/缓慢推进/跟拍/摇移，不叠加}
+
【3-5条关键负面约束】
```

### 五段式提示词示例

**叙事镜（S级）**：

```
【固定主体】主角，男性，白发，粗糙手掌，朴素衣着
【核心动作】弯腰取物，提水出井
【场景/光线】古井旁，黎明前灰蓝雾霭，暖色侧光从东方渗入
【主运镜】固定镜头，从中景缓慢推到近景（3s→6s），不旋转不加速
【负面约束】不要变脸，不要多余手指，不要文字乱码，不要突然换场景，不要物体变色
```

**空镜（C级）**：

```
【固定场景】室外全景，无人出现，石桥流水
【核心动作】无人物动作。水面静止，雾气从水面升起，树叶微动
【场景/光线】清晨薄雾，冷青蓝调，逆光剪影
【主运镜】固定镜头，静止，不做任何运动
【负面约束】不要人物，不要文字，不要建筑变形，水面不要涟漪
```

**过渡镜（B级）**：

```
【首帧参考】上一镜最后一帧（主角端面向门口走）
【尾帧参考】下一镜第一帧（配角坐在桌边等）
【核心动作】主角从灶台走向门口，镜头跟随，物品从左手换到右手
【场景/光线】室内，灶火暖光，窗外晨光渗入

| 污染类型 | 检测正则 | 修复 |

| 运镜叠加 | 同一prompt含≥2种运镜关键词（推+摇+跟+升+降+旋转+环绕·任意2种） | 只保留一种 |
| 动作叠加 | 同一prompt含≥2个核心动作（转身+跑+拔剑+摔倒·任意2种） | 拆成多镜 |
| 光线矛盾 | "白天"+"夜景" / "暖光"+"冷光" / "逆光"+"顺光" 同镜出现 | 只保留一项 |
| 场景跳变 | "室内"+"室外" / "白天"+"黑夜" 同镜出现 | 只保留一个场景 |
| 角色数量 | 同一prompt含≥3个命名角色 | 拆成≤2角色/镜 |
| 约束超量 | 负面约束≥8条 | 精简至≤5条 |

## J.5 单变量修复法（SQ12增强）

> 重生成时**每次只改一个变量**，禁止全量重写prompt。

### 排查顺序（优先级从高到低）

| 问题现象 | 只改哪个变量 | 不动哪些 |
|------|------|------|
| 角色变脸/服装错误 | **回参考图**（重新上传角色三视图） | 不动场景/运镜/动作/光线 |
| 运镜乱飞 | **改运镜**：降级为"固定镜头"或"缓慢推进" | 不动角色/场景/动作 |
| 动作失败/肢体扭曲 | **拆动作**：从复合动作拆成单一动作 | 不动角色/场景/运镜 |
| 场景跳变 | **锁定场景**：加注"保持空间结构一致" | 不动角色/运镜 |
| 光影闪烁 | **固定光源**：指定单一光源方向 | 不动角色/场景/动作 |

### J.5.1 重抽上限

```
单镜重抽 ≤ 3次（超过→人工介入，展示全部3次日志+对照差异列表）
全片重抽 ≤ 1次（超过→展示所有失败记录，等用户决策）
```

【2·场景锁定】地点+空间结构+材质三层+尺度锚点+竖屏七法+光影基调+色调注入%
【3·动作锁定】核心动作+拆解(秒级分段)+速度+幅度+冗余微细节
【4·运镜锁定】主运镜(仅一种)+起幅→落幅
【5·表情精控】方案一程度副词+方案二穷举+方案三分段时序(0-0.5/0.5-1.5/1.5-3/3-5s)+方案四视觉焦点线+方案五生理呼吸+负向约束
【6·六维肌肉】维1肌群键盘(动/不动/交互约束)+维2情绪三层(主/副/动机)
【7·可控瑕疵】1-2类(前景遮挡/空气质感/动态微扰/人群模糊/光衰减/冗余微动作)
【8·交接动作】起始动作承接上镜+结束动作延续下镜+过渡方式
【9·音效层】物理声源+环境声+Foley(⚠️不写BGM/配乐/旋律)
【10·负向约束】不变脸/不换场景/不运镜叠加/不光影跳变/不动作突变
【11·资产关联】角色R?+场景S?+道具?
```

# 附录 L：AI漫剧工业化流水线（28天→7天·LoRA+批量）

> **来源**：AI漫剧生产SOP实测（2026）。核心：先慢后快。第一集搭管线踩坑定标准→后面流水线作业。

## L.1 四步流水线

### 第一步：出分镜（别急着生图）

| 操作 | 工具 | 要点 |
|------|------|------|
| 剧本初稿 | Claude / GPT-4 | 把节奏/对白/冲突打磨顺 |
| 分镜草图 | 豆包 | 导剧本→出分镜草图 |
| **手动拆分镜头** | 人工 | ⚠️ 最关键一步：每个场景需要几个镜头？什么景别？怎么接？ |
| 生成分镜草图 | AI绘图工具 | prompt加"storyboard sketch"不要求精美·构图和感觉对就行 |

### 第二步：出画面（最深的坑）

| 操作 | 工具 | 要点 |
|------|------|------|
| 定风格 | 人工决策 | 真人感/动漫风/3D卡通风→三选一·定了不改。新手建议动漫风（角色一致性好控制/观众容忍度高/出片效率最高）|
| **训练LoRA** | 训练平台 | ⚠️ 核心：5-10张角色图训练LoRA。单角色≈几十元。省百倍返工时间。后面所有图基于此模型生成→角色长相锁定 |
| 精修提示词 | — | 固定特征写入每张图prompt。例："白发稀疏束脑后的70岁男性，灰色粗布衣，深蓝围裙"——每张图必带 |
| 主力出图 | Midjourney悠船版 | 画质和风格稳定性最强 |
| 备选出图 | 即梦 | 出片快但细节差一些 |

### 第三步：做视频（让画动起来）

| 操作 | 要点 |
|------|------|
| 上传图片+写动作指令 | 别让AI自由发挥。越具体越好："他在灶台边缓慢揉面，蒸汽从锅里升腾，金色微粒随之飘浮" |
| 片段规格 | 每片段4-10秒·一集5-15片段（对应5/6镜宏镜头） |
| 组装三注意 | ①镜头逻辑连贯（特写不突然跳全景）②动作接得上（揉面不突然切端碗）③画面内容不断层（道具/服装/场景一致）|
| 主力工具 | 可灵 |
| 备选工具 | 即梦 |

### 第四步：配音剪辑（最后一道关）

| 操作 | 工具 | 要点 |
|------|------|------|
| 智能字幕 | 剪映 | 文本→智能字幕→校对错别字 |
| **多角色分声线** | 剪映 | ⚠️ 核心：主角一个声音·配角换不同声线·旁白再找沉稳的。别让所有角色共用一个声音——一秒出戏 |
| 配BGM | 剪映 | 紧张→低音节奏+鼓点·情感→钢琴/弦乐·搞笑→轻快电子。别全片同一首 |
| 精修节奏 | 剪映 | 画面切换卡音乐节奏点·每段≤10秒·配音口型对齐 |

# 附录 P：场景转场系统（5种核心转场）

> **来源**：Scene Transition Library (ClawHub, beermanzz)。
> **定位**：AI短剧分镜脚本中的镜头转场标注规范。与 M.5 分镜表14字段联合使用。
> **触发**：转场、镜头转换、过渡、场景切换、叠化、硬切、匹配剪辑、淡入淡出。

## P.1 五种转场类型速览

| 类型 | 标注符号 | 适用场景 | 情绪效果 | AI执行难度 |
|:--:|------|------|------|:--:|
| **硬切** | `下一秒` / `画面骤转` | 突发/强调/节奏突变 | 紧张/冲击 | ⭐ 最易 |
| **叠化** | `↔` / `叠化` | 并行场景/时间流逝 | 柔和/诗意 | ⭐⭐ 易 |
| **匹配剪辑** | `→（匹配剪辑）` | 平滑过渡/视觉呼应 | 连贯/巧妙 | ⭐⭐⭐ 中 |
| **淡入淡出** | `淡出` → `淡入` | 情绪收尾/章节分隔 | 终结/新章 | ⭐⭐ 易 |
| **划像** | `←划入` `划出→` | 空间位移/视轴切换 | 动态/转换 | ⭐⭐ 易 |

## P.2 转场标注格式

```
[镜头A描述] → [转场词] → [镜头B描述]
```

**示例：**
- 硬切：`拳头落下，下一秒，画面切至医院走廊`
- 叠化：`古钟特写 ↔ 水面倒影，金色文字浮现`
- 匹配剪辑：`握紧的拳头 →（匹配剪辑）→ 握紧的枪柄`
- 淡入淡出：`灯光熄灭，淡出。淡入，清晨阳光洒入空房间`
- 划像：`←划入 新场景 `

## P.3 转场 Prompt 写作原则

1. **转场词前置**：先写「从哪里来」，再写「到哪里去」
2. **意象连接**：上下镜头的意象元素有视觉或语义上的呼应
3. **情绪一致性**：转场方式与叙事情绪匹配——快速转场=紧张，缓慢叠化=情绪铺垫
4. **AI适配**：转场描述简洁具体，不混合多种转场类型在同一标注中
5. **竖屏适配**：划像需留足屏幕边缘空间，9:16构图下优先硬切和叠化

## P.4 短剧转场选择速查

| 场景切换类型 | 推荐转场 | 理由 |
|------|:--:|------|
| 同场景时间跳跃（几小时后） | 叠化 | 时间流逝感 |
| 突发危机/意外 | 硬切 | 制造冲击 |
| 回忆/闪回 | 淡入淡出 | 时间分界 |
| 双线叙事切换 | 匹配剪辑 | 视觉呼应 |
| 集尾/章节结束 | 淡出 | 情感收束 |

> **与短剧大师关联**：转场标注嵌入分镜表（M.5第8字段「角色动作」中），成为SQ8镜头衔接检测的一部分。

# 附录 Q：时序连贯性锚点法则（7大法则·实用摘要）

> **来源**：《AI视频生成电影级连贯性核心技术白皮书》（PixelFlow，2026）。
> **定位**：从理论层到prompt层的连贯性保证体系。原文含完整数学推导，本摘要提取可直接用于短剧创作的实用规则。

## Q.1 七大法则速览

| # | 法则 | 一句话 | prompt层落地 |
|:--:|------|------|------|
| ① | **运动连续性锚点** | 相邻帧的运动轨迹必须可追溯 | prompt加：「运动轨迹平滑，无跳变，动作从上一帧自然延续」 |
| ② | **隐空间稳定性锚点** | 隐状态不能突变，否则画面闪烁 | 加负面词：「no flickering, stable consecutive frames」 |
| ③ | **关键帧语义锚定** | 同一概念跨帧时特征一致 | prompt加：「角色面部特征、服装、发型全程保持一致」 |
| ④ | **场景流一致性锚点** | 深度与运动耦合，远近关系不崩 | prompt加：「景深关系稳定，前景与背景的相对位置不突变」 |
| ⑤ | **时序残差传播锚点** | 帧间变化量可微分控制 | 加控制词：「gradual movement, incremental change, no sudden jumps」 |
| ⑥ | **运动学连贯性锚点** | 角色动作符合物理规律 | 加约束：「natural body mechanics, physically plausible motion」 |
| ⑦ | **叙事逻辑锚点** | 镜头序列与剧本图谱拓扑一致 | SQ4检测：镜头顺序=大纲事件顺序，无遗漏无倒置 |

## Q.2 短剧连贯性常见故障及修复

| 故障现象 | 根因（锚点失效） | Prompt修复 |
|------|:--:|------|
| 画面闪烁/抖动 | ②隐空间不稳定 | 加 `stable consecutive frames, consistent lighting, no flicker` |
| 角色换脸/变装 | ③关键帧语义漂移 | 加 `facial features, hairstyle, and outfit remain identical across all frames` |
| 动作断裂/瞬移 | ①运动不连续 | 用交接动作公式（J.7）：`保持奔跑惯性，逐渐减速停下` |
| 空间关系紊乱 | ④场景流不守恒 | 加 `depth of field consistent, background parallax smooth, spatial relationships stable` |
| 镜头逻辑错误 | ⑦叙事逻辑断裂 | SQ4审查：大纲事件逐项核对，确保每事件有对应镜头 |
