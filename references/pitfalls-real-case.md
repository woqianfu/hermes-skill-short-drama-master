## 批量审查方法论

### 审查架构：三路并行 delegate_task

```
60集 → 3批（每批20集）× 3个子代理并行
  Batch A: E01-E20   (角色建立/世界观)
  Batch B: E21-E40   (代价曲线/反派升级)
  Batch C: E41-E60   (高潮/终局一致性)

每个子代理按 SQ0-SQ12 逐集审查，回报：
  逻辑问题 + 平台兼容问题 + 格式问题
```

### 子代理的审查指令模板

```
审查标准（基于短剧大师 v5.9 SQ0-SQ12）：

一、逻辑错误检查（SQ4+SQ5）：
1. 角色外表/服装/道具在连续集之间一致性
2. 场景空间锚定物位置/尺寸是否前后一致
3. 暗线推进：封印/代价/灵脉状态是否连贯
4. 前集尾钩→下集回应是否到位
5. 时间线：日期/时辰标注是否一致

二、小云雀平台兼容性（SQ6+SQ8）：
1. 每镜prompt是否包含必要元素（主体+动作+场景+光线+运镜+负向约束）
2. 是否有小云雀NLP分词器无法解析的句式
3. 材质/尺度锚点/光影基调/竖屏七法字段是否完整

三、格式规范性：
1. Phase 1元数据完整性
2. 暗线/武打/尾钩/爆款节奏表统计准确性

重点：be critical and specific。每个问题必须引用具体行号或内容。
```

## 发现的高频错误模式

### 模式1：封印完整度回退（🔴 致命）

**症状**：封印已在前集突破（如 E04: 7→6/7），但后续集数中标注回退了（E06-E09 全部标 7/7）。

**检测方法**：
```bash
# 提取所有封印标注，按集排序检查
grep -n "封印完整度\|封印：\d" E*.md | sort
```

**根因**：不同集可能由不同时间创作，中间状态的遗忘。

**修复**：逐集校正，维持单调递减（封印只能变弱不能变强，除非有明确剧情解释）。

**本次案例**：E04 封印突破至 6/7，E05 保持 6/7，但 **E06-E09 全部错标为 7/7**（4集连续4处错误），E10 又从 7→6。修正后 E06-E09 统一为 6/7。

---

### 模式2：代价百分比V型曲线（🔴 致命）

**症状**：代价数字出现不可能的回升（数字变大=代价减轻，但剧情中代价在恶化）。

```
错误示例：
100%(E01)→97%(E04)→75%(E05)→100%(E06)←不可能回升
```

**根因**：代价百分比未统一定义（是"剩余健康度"还是"已消耗量"？），且各集写作时未参照前集数字。

**检测方法**：
```bash
# 提取所有代价标注，检查单调性
grep -oh "代价.*[0-9]*%\|代价：[0-9]*%" E*.md | sort
```

**修复**：在剧本总纲中明确定义：「代价X% = 延炎体内可用灵脉储备剩余百分比。100%=完整修为，0%=修为枯竭→死亡。」然后逐集重校数字。

**本次案例**：E01-E10 代价形成不可能曲线（100→97→75→100→97→75），修正为 100→97→90→88→85→75。

---

### 模式3：版本断裂/硬拼接（🔴 致命）

**症状**：某集之后，角色团、世界观体系、代价机制突然全变，无任何过渡。

**检测方法**：
```bash
# 检查每集出场角色列表是否一致
grep -n "在场人物\|出场角色\|五人\|六人" E[4-5]*.md
```

**本次案例**：E41-E43 使用六人团（延炎/林依依/张猎户/孙九/小鹿/赵铁匠）+ 锁妖环体系 + 逐步退化代价线；E44-E60 突然切换为五人团（江暮寒/苏青禾/洛无伤/赤牙替换四人）+ 苍生塔体系 + 一次性献祭代价线。**必须二选一重写其中一段。**

---

### 模式4：破折号断裂句式（🔴 小云雀致命）

**症状**：叙事文本中每个词汇用「——」分隔：

```
延炎——右手按在井沿——金焰——顺右臂——涌——入井
```

**为什么小云雀无法解析**：
- NLP分词器把每个「——」当分隔符 → Token被切碎
- 「延炎」「右手」「按在井沿」各自成为独立token → 语义依存关系丢失
- AI无法判断「谁」「做了什么」「在哪里」

**检测方法**：
```bash
# 统计每集破折号密度
grep -c '——' E*.md | sort -t: -k2 -rn | head
```

**修复方法**（三阶段处理）：

1. **替换阶段**：将词汇级「——」替换为逗号 `，`
2. **合并阶段**：将逗号分隔的单字碎片重新合并为词语（`暗，金，光` → `暗金光`）
3. **保护阶段**：保留结构性标注（【镜N】【竖屏七法】【光影基调】【暗线A-D】）

修复后：
```
延炎的右手按在井沿，金焰顺着右臂涌入井中。
```

**本次案例**：E26-E43 共18集，总计约 **4万处** 破折号替换为中文逗号，叙述正文**零残留**破折号断裂句式。

---

### 模式5：代价大跳斜率异常（🟡 中等）

**症状**：代价在一集内跳变过大（如单集降 8%），但无前期身体恶化预兆。

| 正常斜率 | 异常斜率 | 修复 |
|----------|----------|------|
| 每集降 1-2% | 单集降 5-8% | 在前集尾钩增加预兆 + 孙九代价曲线更新预测 |

**本次案例**：E25(-8%)、E30(-6%)、E35(-5%)、E40(-3%) 四处大跳需补入「骨头在凉」「金痕异常加速」等身体预兆描写。

---

### 模式6：Prompt必需字段退化（🟡 中等）

**症状**：前期集（E01-E25）每镜都有完整标注（竖屏七法/光影基调/材质三层/尺度锚点/运镜），但后期集逐渐弱化或完全缺失。

**检测方法**：
```bash
# 检查关键字段是否存在
grep -L "竖屏七法" E*.md
grep -L "尺度锚点" E*.md
```

**本次案例**：E44-E60 共16集**完全缺失** prompt 级描述，文学性叙事无法直接用于小云雀生成。

---

### 模式7：关键世界观信息过晚揭示（🟡 中等）

**症状**：核心设定（封印结构/灵力体系/隐藏角色）集中在后期集数一次性抛出，前期"不知原理看谜团"持续过长。

| 信息 | 揭示集 | 建议提前到 |
|------|--------|-----------|
| 封印两层结构 | E19 | E04/E10 |
| 五种灵力类型 | E19 | E04→E10分散 |
| 第三片锁/另一人 | E20 | E16 |
| 穆长老半壳=锁妖环 | E38 | E25暗示 |
| 月白/天工族 | E34 | E28三角分析时 |

## 审查流水线检查清单

执行批量审查时，按以下顺序检查：

```
□ SQ0：文件版本确认（AIGC正版 vs 旧版 vs 时间戳副本）
□ SQ1：角色外观/服装/道具连续性
□ SQ2：场景空间锚定物一致性
□ SQ3：暗线四线状态单调性（封印↓/代价↓/灵脉单向变化）
□ SQ4：跨集叙事逻辑（尾钩→回应链）
□ SQ5：时间线标注一致性
□ SQ6：每镜prompt必需字段（竖屏七法/光影基调/材质/尺度锚点/运镜/负向约束）
□ SQ7：破折号断裂句式检测
□ SQ8：代价大跳斜率合理性
□ SQ9：版本断裂检测（角色团成员是否一致）
□ SQ10：关键信息揭示时机是否过晚
```

## 修复流水线

### 标注类修复（封印度/代价/暗线）→ sed批量替换
```bash
sed -i '' 's/封印完整度：7\/7/封印完整度：6\/7/g' E0[6-9]*.md
```

### 句式类修复（破折号断裂）→ delegate_task三路并行
```
18集 → 3批（每批6集）× 3个子代理
  每个子代理：读取→识别断裂→替换合并→保护结构性标注→写回
```

### 版本断裂 → 明确方向后大规模重写
需用户确认以哪个版本为准，然后对应的20集重写。
