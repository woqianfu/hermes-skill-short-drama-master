# 大师知识吸收协议 — 当用户说「让XX大师学习一下XXX」

> **定位**：每次用户带来外部知识源（公众号文章/信息图/教程/论文），希望吸收到某位大师时的标准操作流程。
> **来源**：实战提炼 — 灯光大师吸收40种光影 + 运镜大师吸收16种叙事构图 (2026.06.30)
> **触发关键词**：「让XX大师学习一下」「XX大师学这个」「把XX教给XX大师」
> **关联**：Phase 2.5 排期会的知识前置环节；进化大师KPI（知识吸收完成率）

---

## 一、判定标准

| 用户说 | 含义 | 行动 |
|:-------|:-----|:-----|
| 「让XX大师学习一下XXX」 | **启动吸收协议** | 执行八步法 |
| 「看看这个」 | 需要先确认 | 先分析内容→判断归属大师→问用户「吸收到XX大师？还是仅供参考？」 |
| 「把XX教给XX大师」 | 明确吸收指令 | 直接执行八步法 |

## 二、吸收八步法

```
用户提供外部知识源
  │
  ├─ 第1步：解析知识源
  │    提取核心内容 → 确定属于哪个大师的知识领域
  │    分类：①提示词模板类（灯光/运镜/构图）
  │          ②方法论类（武术/节奏/暗线）
  │          ③工艺类（省钱/转场/废片）
  │          ④法规类（审核）
  │
  ├─ 第2步：创建参考文件
  │    写入 `references/<topic-name>.md`
  │    格式：源信息（来源+日期）+ 定位说明 + 分类内容 + 速查表
  │    文件名用英文+数字，全小写，短横线连接
  │    每种知识配中文标题 + 效果说明 + 可直接复制的英文/中文提示词
  │
  ├─ 第3步：更新大师一句话描述
  │    在22位大师封号表中，追加新知识的简介 + 参考文献名
  │    例：灯光大师→「40种漫剧光影提示词模板库（参考文献O.8）」
  │
  ├─ 第4步：更新会审标准
  │    在二十二位大师会审标准表中，追加新的审查焦点和输出格式
  │    例：运镜大师→加「叙事构图符合人物情绪」+ `[叙事构图]`
  │
  ├─ 第5步：更新模块引用表
  │    在SQ总览下方的模块表中追加一行
  │    格式：| **知识名称** 🆕 | 内容说明+搭配关系 | `references/文件名.md` |
  │
  ├─ 第6步：更新prompt组装参考
  │    在第3.5节prompt组装参考中，追加新知识的引用路径
  │
  ├─ 第7步：更新排期会输入
  │    在Phase 2.5排期会中，更新对应大师的输出字段+引用路径
  │
  ├─ 第8步：更新references/index.md
  │    在文件索引中添加新文件条目，标注⭐和来源信息
  │
  └─ 完成后告知用户「XX大师已学会 ✅」
```

## 三、参考文件格式规范

每个 `references/<topic>.md` 必须包含以下头信息：

```markdown
> **来源**：{作者/公众号}《{标题}》（{日期}）
> **吸收日期**：{YYYY-MM-DD}
> **定位**：{XX大师}的{知识领域}补充。{一句话说明与现有体系的关系}
> **适用**：{在哪个环节使用}
```

### 内容结构要求

| 章节 | 必含 | 说明 |
|:----|:----:|:-----|
| 速览表 | ✅ | 表格形式，一列名称一列核心意图 |
| 详细条目 | ✅ | 每条配：效果说明 + 提示词 + 适配情节 + 搭配建议 |
| 速查表 | ✅ | 按情绪/场景反向检索 |
| 与现有体系对照 | ✅ | 说明本库与现有竖屏七法/20种运镜/附录O等的关系 |
| 冲突风险 | ✅ | 涉及跨大师冲突时标注（灯光vs运镜、构图vs运镜等） |

### 提示词格式规范

```markdown
### {编号}·{名称}
**效果说明**：{一句话描述光效/构图的效果}
**提示词**：`{英文关键词，用逗号分隔，可直接复制到prompt}`
**适配情节**：{适用剧情类型}
**运镜搭配**：{←引用附录J.3.2的运镜编号}
**灯光搭配**：{←引用附录O的光影编号}
```

## 四、无需启动吸收协议的场景

- ❌ 纯参考信息（不涉及任何大师的知识库扩展）
- ❌ 用户自己的笔记或备忘录（写入MEMORY.md或session即可）
- ❌ 一次性分析需求（如「分析这个案例」→ 案例大师在会话中分析即可）
- ❌ 跨大师协作问题（→ 走Phase 2.5排期会流程，不走吸收协议）
