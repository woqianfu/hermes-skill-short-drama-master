# 小云雀/API提交审批规则

> 写于2026年6月18日，因同一错误犯三次后强制写入。
> 位置：短剧大师 references + memory + 桌面铁律.md 三处写死。

## 铁律

1. **只有"跑"字算数。** "试试"/"看看"/"发一下"/"你操作"/"你看着办" 都不算批准。
2. **每一步都要问。** 提交→轮询→下载→生成，中间不能跳步。上一步批了≠下一步自动批。
3. **模型/参数必须确认。** 用户说seedance2.0_mini就是seedance2.0_mini，不准推断/换/补。
4. **提交前先过会审。** 剧本必须先经过短剧大师SQ质量关卡审查，再提交小云雀。
5. **优先用浏览器提交。** 用户能在网页看到过程，而不是CLI黑箱操作。

## 正确流程

```
用户说"跑" → 先过SQ审查 → 问用户准不准提交
用户说"跑" → submit_run → 展示链接，停下
用户说"看看" → 轮询
用户说"下载" → 下载
```

## 已犯过的错误（不准再犯）

```
❌ 用户说"试试" → 自己自动提交+轮询+下载一条龙
❌ 用户说seedance2.0_mini → 自己换成seedance2.0_fast_vision
❌ 提交后不等话 → 自动轮询
❌ 书面承诺三次 → 继续犯
```
